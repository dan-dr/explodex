# Explodex

## Register

product

## Users

Codex desktop users and plugin authors who want focused, high-leverage workflow improvements without replacing the official app. They work in long-running coding sessions, often across several projects and threads, and need extensions to feel native and stay out of the way.

## Product Purpose

Explodex extends the official Codex desktop app through a local plugin SDK. Success means useful workflows can be added, removed, and iterated independently while preserving Codex's interaction model and leaving the installed application untouched.

## Brand Personality

Native, pragmatic, precise. The interface should feel like a capability Codex already had: compact, legible, calm, and operationally clear.

## Anti-references

- Standalone dashboard aesthetics that visibly diverge from Codex.
- Decorative plugin chrome, excessive cards, or novelty motion.
- Hidden or surprising behavior that obscures which thread, project, or tool is active.
- Fragile visual approximations where Codex-native controls or tokens are available.

## Design Principles

1. Extend Codex; do not visually compete with it.
2. Keep the active task and its state obvious.
3. Prefer direct manipulation for spatial workflows and explicit recovery for destructive actions.
4. Make advanced capability discoverable without adding permanent clutter.
5. Treat reverse-engineered integration points as failure boundaries with clear fallbacks.

## Accessibility & Inclusion

Preserve Codex's keyboard and focus conventions. New controls need visible focus, descriptive labels, sufficient contrast, reduced-motion behavior, and pointer targets appropriate for dense desktop UI.
